# 微服务认证授权说明

## 1. 架构概览
本项目采用基于 **Spring Cloud Gateway + JWT + Redis** 的统一认证方案。

```
[Gateway] (Port: 8080)
    |
    |-- [AuthGlobalFilter] (Priority: -1)
    |   |-- 1. 白名单检查 (Skip Auth)
    |   |-- 2. 解析 Authorization Header (JWT)
    |   |-- 3. 校验 JWT 签名与有效期
    |   |-- 4. 提取 UserId/Role 注入请求头 (X-User-Id, X-User-Role)
    |
    |-- [Business Services] (e.g., auth-service, business-service)
        |-- 1. 从请求头读取 X-User-Id/X-User-Role
        |-- 2. 业务逻辑处理
```

## 2. 核心模块
- **common-service**: 提供 `UserContext` 工具类、`@IgnoreAuth` 注解、统一 `Result<T>` 格式。
- **auth-service**: 负责登录（生成 JWT）、登出（注销 Redis 缓存）、用户基础校验。
- **gateway-service**: 统一网关，负责全局鉴权、跨域配置、限流控制。

## 3. 配置说明
### 3.1 配置文件
- `config/application.yml`: 主配置文件，包含 Nacos 注册中心与配置中心的地址，以及引用的共享配置文件。
- 所有具体的数据库、Redis 和 RabbitMQ 配置均托管在 Nacos 中，具体数据可参考 `docs/nacos-config.md`。

### 3.2 敏感信息加密 (Jasypt)
为了保证安全性，配置文件中的密码均使用了 Jasypt 进行加密。
- 加密密钥为：`xuanyuan-secret`
- 在配置文件中，加密内容以 `ENC(...)` 形式展现。

## 4. 开发指南
### 4.1 获取当前登录用户
在业务微服务中，可以直接调用 `UserContext.getUserId()` 获取当前用户的 ID。网关会自动解析 Token 并透传到请求头中。

### 4.2 跳过鉴权
- **白名单配置**: 在 `gateway-service` 的 `application.yml` 中配置 `auth.whitelist.urls`。
- **自定义注解**: 在 Controller 或方法上标注 `@IgnoreAuth`。

## 5. Postman 调试场景
详细配置请参考 `docs/OnlineLearning_Postman_Collection.json`。
- **场景1：登录** -> 获取 Token 并存入环境变量。
- **场景2：访问受保护接口** -> 自动携带 `Authorization: Bearer {{token}}`。
- **场景3：访问白名单接口** -> 无需 Token 即可访问。
- **场景4：登出** -> 注销 Token 并清除 Redis 缓存。
