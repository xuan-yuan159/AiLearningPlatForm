# 课程管理微服务 (course-service) 说明

## 1. 模块定位
`course-service` 主要负责课程资源的核心业务管理，面向教师端提供课程创建、编辑、上下架、章节规划与资源绑定等能力；同时为用户端提供课程信息检索与查询服务。

## 2. 数据层设计
严格遵循 `schema.sql`，实体层主要操作：
- `course`: 课程主表
- `chapter`: 章节表
- `resource`: 视频/资料等资源表

> ⚠️ 注意：基于规定，不对已有表结构增加字段，软删除采用更新 `status = 3` 实现（或在前端视角 `2` 为下架并作删除使用，本服务目前手动将 `status` 置为 3 标示已删除）。

## 3. 技术栈与中间件复用
- **Web框架**: Spring Boot 3.2.x + Spring Cloud Alibaba 2023
- **ORM**: MyBatis-Plus 3.5.x，全局禁用原生 SQL 编写。
- **缓存 (Redis)**: 
  - 使用 `StringRedisTemplate` 进行缓存控制。
  - **课程详情**: Key 格式为 `course:{courseId}`，过期时间 `10 分钟`。
  - **课程列表**: Key 格式为 `course:list:{categoryId}:{page}`，过期时间 `10 分钟`。
- **消息队列 (RabbitMQ)**: 
  - 课程状态变更（发布/下架/删除）时，通过封装在 `common-service` 的 `EventPublisher` 发布 `CourseStatusChangeEvent` 事件。
  - Topic 约定为 `course.status.change`，以便其它服务（如统计、搜索）订阅消费。
- **公共组件**: 
  - `UserContext`: 获取请求头的用户身份
  - `Result`: 统一返回包装
  - `GlobalExceptionHandler`: 全局异常捕获
  - OpenAPI 3 注解：为所有 API 接口声明 Swagger 分组与参数说明。

## 4. API 接口预览
服务默认暴露在 `8082` 端口（实际根据 Nacos 配置与网关路由调用）。
可在 `http://localhost:8082/doc.html` 查看完整的 Swagger 接口文档。
