## 项目运行说明

### 1. 项目结构与技术栈
- 后端：Spring Boot（Java 17）
  - 目录：`Backend/OnlineLearning`
- 前端：Vue 3 + Vite
  - 目录：`Frontend/OnlineLearning`
- 数据库：MySQL 8.x
  - 建表脚本：`Others/schema.sql`
- 缓存：Redis
- 消息队列：RabbitMQ（可选，取决于后端实现/接口是否启用）

### 2. 运行前准备
本项目统一使用**远程服务器**上的基础设施，不需要本机再单独安装 MySQL / Redis / RabbitMQ / Nacos，也不需要本地修改配置文件。

本地仅需准备：
- JDK 17
- Maven（用于构建/运行 Spring Boot）
- Node.js（前端使用 `package.json` 指定的 Node 版本范围）

### 5. 启动方式（开发模式）
#### 5.1 启动后端
```bash
cd Backend/OnlineLearning
mvn spring-boot:run
```
后端默认端口通常是 `8080`（如需自定义可增加 `server.port=xxxx`）。

#### 5.2 启动前端
```bash
cd Frontend/OnlineLearning
npm install
npm run dev
```
前端默认端口通常是 Vite 的 `5173`，打开终端输出的 URL 即可访问。

### 6. 前后端联调说明（当前状态需要注意）
前端的接口层在多个 API 文件中仍然携带了 `mockData`，因此当前“请求不会真正发到后端”，页面即使后端接口缺失也可能正常展示（mock 返回数据）。

如果你希望切换为“真实后端联调”，通常需要：
1. 去掉 `mockData` 参数，让 `request()` 走真实 `fetch`
2. 解决 `/api/...` 路径的代理问题
   - 前端目前使用的是相对路径（例如：`/api/teacher/...`），本地开发时会先请求到 `http://localhost:5173/api/...`
   - 需要在 `vite.config.js` 配置 `server.proxy`，或使用 Nginx 进行反代

生产部署反代示例可参考：
`Frontend/OnlineLearning/docs/部署指南.md`

### 7. 常见问题排查
- 后端启动后前端仍“登录/接口无响应”
  - 检查前端是否仍在 mock 模式（看 `src/api/*Api.js` 是否传了 `mockData`）
  - 如果已关闭 mock，请检查是否完成 `/api` 代理或 Nginx 反代
- 连接 MySQL 失败
  - 检查 `spring.datasource.url/username/password` 与数据库权限
- 连接 Redis/RabbitMQ 失败
  - 检查 host/port/密码及防火墙/安全组策略
