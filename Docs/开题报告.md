## 📝 摘要

随着信息技术的飞速发展与混合式学习模式的常态化，在线教育已成为现代教育体系的重要支柱。然而，当前部分在线学习平台多侧重于课程资源的静态展示，普遍存在师生互动滞后、学习反馈不及时、个性化服务不足等问题，难以满足学生日益增长的智能化学习需求。与此同时，大语言模型（LLM）技术的突破性进展为教育领域的智能化转型提供了新的契机。

本课题拟设计并实现"基于SpringBoot+Vue的AI助教在线学习平台"。系统采用前后端分离架构，后端基于Spring Boot微服务框架，前端利用Vue 3构建响应式界面，确保系统的高可用性与易维护性。核心创新在于集成第三方大语言模型API，通过合理的提示词工程与上下文管理机制，实现课程自动摘要生成、基于语义理解的智能答疑、个性化学习计划动态推荐等AI助教功能。同时，引入Redis缓存机制优化热点数据访问性能，采用JWT令牌认证保障用户会话安全，结合MySQL数据库索引优化策略，全面提升系统响应速度与稳定性。

本研究不仅顺应智慧教育发展趋势，也为前沿AI技术与成熟Web框架的结合落地提供了可行的技术方案，具有一定的理论意义与应用价值。

## 一、选题意义

### 1.1 技术应用价值

- **架构设计**：后端基于Spring Boot微服务架构，前端利用Vue 3构建响应式界面，确保系统高可用性与易维护性
- **AI集成**：集成第三方大模型API，实现课程自动摘要、智能答疑及个性化计划制定
- **性能优化**：利用Redis缓存与JWT认证保障高并发性能与安全
- **技术探索**：探索前沿AI技术与成熟Web框架结合，对智慧教育落地具参考意义

### 1.2 社会实践价值

- AI助教提供 7×24 小时辅导，提升学生学习效率
- 自动化管理减轻教师负担
- 涵盖软件工程全生命周期，综合运用需求分析、架构设计及全栈开发知识
- 在AI接口整合与性能优化环节锻炼工程实践能力

## 二、文献综述

### 2.1 国内研究现状

- 在线教育平台数量激增，但多侧重于课程资源的数字化存储与基础管理
- 智能化交互方面仍显不足，多基于规则引擎实现简单推荐
- 缺乏对学生学习状态的深度语义理解
- 部分研究探索基于知识图谱的答疑系统，但维护成本高、灵活性差
- Spring Boot+Vue前后端分离架构已证明在提升系统可维护性方面的优势

### 2.2 国外研究现状

- 智能化在线教育起步较早，部分平台已尝试机器学习算法进行课程推荐和作业自动评分
- 自适应学习系统通过数据分析动态调整学习路径，显著提升学习效率
- 国际知名教育平台推出的智能助教利用大模型技术提供引导式教学
- 高级AI功能通常依赖昂贵的云计算资源，难以在普通院校普及
- 数据隐私在AI教育应用中的重要性需加强权限控制与数据加密

### 2.3 研究问题总结

1. 智能化程度低，师生互动滞后，缺乏即时反馈机制
2. 技术架构陈旧，难以支撑高并发与AI接口的快速集成
3. 个性化不足，无法根据学生实际情况生成动态学习计划
4. 微服务架构与大模型API深度结合构建低成本AI助教的研究相对较少

## 三、设计任务（研究内容）

### 3.1 系统需求分析与总体架构设计

- 调研国内外在线教育平台现状及师生实际需求
- 完成功能性需求与非功能性需求分析
- 明确学生、教师及管理员三种角色的权限边界与业务流程
- 设计前后端分离的系统总体架构：
  - 后端：Spring Boot + Spring Cloud微服务架构
  - 前端：Vue 3 + Vue Router实现SPA路由管理
- 制定统一的RESTful API接口规范

### 3.2 核心业务功能模块的设计与实现



| 模块             | 功能描述                                                     |
| ---------------- | ------------------------------------------------------------ |
| **用户管理模块** | 用户注册、登录及个人信息管理；采用JWT实现无状态用户认证      |
| **课程管理模块** | 教师端：课程发布、编辑、章节管理及资源上传；学生端：浏览课程详情、在线选课及视频学习 |
| **学习进度模块** | 记录学生学习轨迹（视频观看进度、章节完成情况等），为个性化服务提供数据支撑 |

### 3.3 基于大模型的AI助教功能集成研究（核心创新点）



| AI功能                 | 实现方案                                                     |
| ---------------------- | ------------------------------------------------------------ |
| **课程内容自动摘要**   | 利用大模型对长文本课程资料进行语义分析，自动生成课程大纲与核心知识点摘要 |
| **智能答疑系统**       | 构建基于上下文理解的问答接口，调用大模型API生成准确解答，解决师生互动滞后问题 |
| **个性化学习计划制定** | 根据学生历史学习数据与目标，利用AI算法动态生成个性化学习路径与建议计划 |

### 3.4 数据库设计与系统性能优化

- 设计合理的MySQL数据库表结构，绘制E-R图，确保数据一致性与完整性
- 引入Redis缓存机制，对热点课程数据、用户会话等信息进行缓存处理
- 研究数据库索引优化与SQL查询优化策略，提升系统响应速度

### 3.5 系统测试与验证

- 功能测试：单元测试、集成测试
- 性能测试：针对AI接口响应时间的压力测试
- 稳定性验证：不同网络环境下的系统稳定性测试
- 确保AI助教功能的准确性与时效性
