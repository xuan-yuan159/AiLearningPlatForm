# 接口说明文档

本文档用于说明 AI 助教在线学习平台的后端接口协议。

## 全局状态码说明

| 状态码 | 说明 |
| :--- | :--- |
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未授权或 Token 已过期 |
| 500 | 服务器内部错误 |

---

## Knife4j 接口文档

本项目集成了 Knife4j，提供了更美观、易用的接口文档界面。

### 访问地址

| 服务 | 端口 | 访问地址 |
| :--- | :--- | :--- |
| **全局api接口 (gateway-service)** | 8080 | `http://localhost:8080/doc.html` |
| **认证服务 (auth-service)** | 8081 | `http://localhost:8081/doc.html` |
| **课程管理服务 (course-service)** | 8082 | `http://localhost:8082/doc.html` |
| **上传服务 (upload-service)** | 8083 | `http://localhost:8083/doc.html` |

### 调试说明

1. **界面功能**：左侧菜单可以按模块查看接口，支持搜索和离线导出文档。
2. **在线调试**：在接口详情页切换到“调试”页签，填入参数即可发送请求。
3. **全局参数**：可以在右上角的“个性化设置”或“全局参数设置”中统一配置 `Authorization` 请求头。

## 1. 认证模块 (Auth Module)

### 1.1 用户注册

**接口地址**：`/auth/register`

**请求方式**：`POST`

**接口描述**：新用户注册账号。默认注册身份为学生。

#### **请求参数 (JSON)**

| 参数名 | 类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| username | String | 是 | 用户名（唯一） |
| password | String | 是 | 密码 |
| identity | Integer | 否 | 身份（0-学生，1-教师），默认为 0 |
| nickname | String | 否 | 昵称 |
| email | String | 否 | 邮箱 |
| phone | String | 否 | 手机号 |

**请求示例**：
```json
{
    "username": "testuser",
    "password": "password123",
    "identity": 0,
    "nickname": "小明",
    "email": "test@example.com"
}
```

#### **响应参数 (JSON)**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| code | Integer | 状态码（200 表示成功） |
| message | String | 提示信息 |
| data | Object | 响应数据主体（注册成功为 null） |

**响应示例**：
```json
{
    "code": 200,
    "message": "success",
    "data": null
}
```

---

### 1.2 用户登录

**接口地址**：`/auth/login`

**请求方式**：`POST`

**接口描述**：用户通过用户名和密码进行身份验证。验证成功后，系统将返回用于后续请求鉴权的 JWT Token。

#### **请求参数 (JSON)**

| 参数名 | 类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

**请求示例**：
```json
{
    "username": "admin",
    "password": "password"
}
```

#### **响应参数 (JSON)**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| code | Integer | 状态码（200 表示成功） |
| message | String | 提示信息 |
| data | Object | 响应数据主体 |
| └─ token | String | 身份验证令牌 (JWT) |
| └─ userId | Long | 用户唯一标识 ID |
| └─ role | String | 用户角色 (如: STUDENT, TEACHER, ADMIN) |

**响应示例 (成功)**：
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "token": "eyJhbGciOiJIUzUxMiJ9...",
        "userId": 100,
        "role": "ADMIN"
    }
}
```

**响应示例 (失败)**：
```json
{
    "code": 400,
    "message": "Username and password are required",
    "data": null
}
```

---

### 1.3 用户登出

**接口地址**：`/auth/logout`

**请求方式**：`POST`

**接口描述**：注销当前用户的登录状态，并清除服务端（Redis）中的 Token 缓存。

**请求头要求**：
- `Authorization`: `Bearer {token}`

**响应示例**：
```json
{
    "code": 200,
    "message": "success",
    "data": null
}
```

## 5. 上传模块 (Upload Module)

### 5.0 推荐上传流程（新版）

1. 调用 `POST /upload/tasks` 创建上传任务，拿到 `taskId`。  
2. 建立 WebSocket：`/upload/ws/progress?uploadTaskId={taskId}&token={JWT}`，实时接收 `UploadTaskInfo`。  
3. 调用统一上传接口 `POST /upload/resources` 提交文件（推荐在 URL 上追加 `uploadTaskId` 查询参数，便于后端采集“接收阶段”进度）。  
4. 可选：页面刷新后通过 `GET /upload/tasks/{taskId}` 或 `GET /upload/tasks/recent` 恢复进度。  
5. 需要中止时调用 `POST /upload/tasks/{taskId}/cancel`。

---

### 5.1 上传状态机约定

`created`（任务已创建）→ `uploading`（文件接收中）→  
`received`（后端已接收文件）→ `oss_uploading`（上传 OSS 中）→  
`persisting`（保存资源记录中）→ `success`（上传成功） / `error`（失败） / `canceled`（已取消）

> 说明：  
> - 浏览器到后端接收阶段通常展示 `0% ~ 90%`。  
> - OSS 上传与落库阶段通常展示 `90% ~ 100%`。  
> - 后端已支持自动采集进度并通过 WebSocket 推送；`PATCH /upload/tasks/{taskId}/progress` 仅保留兼容。

---

### 5.2 创建上传任务

**接口地址**：`/upload/tasks`

**请求方式**：`POST`

**接口描述**：上传前先创建任务，返回 `taskId`。服务端会校验课程归属与 `resourceType` 合法性。

#### **请求参数 (JSON)**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| courseId | Long | 是 | 课程ID（必须属于当前教师） |
| title | String | 是 | 资源标题 |
| resourceType | String | 是 | 资源类型：`video` 或 `image` |
| fileName | String | 否 | 文件名 |
| fileSize | Long | 否 | 文件大小（字节） |

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "5f2fdc1fd15e421a91db947616f8d2b1"
  }
}
```

---

### 5.3 上传进度 WebSocket 订阅（新增）

**接口地址**：`/upload/ws/progress?uploadTaskId={taskId}&token={JWT}`

**请求方式**：`WebSocket`

**接口描述**：建立连接后，服务端会先推送一次当前任务快照，后续在状态变化时持续推送 `UploadTaskInfo`。

#### **请求参数 (Query)**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| uploadTaskId | String | 是 | 上传任务ID |
| token | String | 是 | JWT；网关会校验并注入用户上下文 |

**推送消息示例**：
```json
{
  "taskId": "5f2fdc1fd15e421a91db947616f8d2b1",
  "teacherId": 1001,
  "courseId": 2001,
  "title": "第1讲-线性回归",
  "resourceType": "video",
  "status": "oss_uploading",
  "stage": "上传到 OSS 中",
  "percent": 94,
  "loadedBytes": 1024000,
  "totalBytes": 1024000,
  "resourceId": null,
  "url": null,
  "objectKey": null,
  "errorMessage": null,
  "createdAt": "2026-04-21 18:00:00",
  "updatedAt": "2026-04-21 18:00:08"
}
```

### 5.4 统一资源上传（主接口）

**接口地址**：`/upload/resources`

**请求方式**：`POST`

**接口描述**：统一处理视频/图片上传，内部完成 OSS 上传与资源落库，返回任务最新状态。

> 建议：调用时使用 `POST /upload/resources?uploadTaskId={taskId}`（同时可在 `form-data` 中传同名字段），可让后端过滤器更准确采集“文件接收中”进度。

#### **请求参数 (`multipart/form-data`)**

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| file | File | 是 | 上传文件 |
| courseId | Long | 是 | 课程ID |
| title | String | 是 | 资源标题 |
| resourceType | String | 是 | 资源类型：`video` 或 `image` |
| uploadTaskId | String | 是 | 上传任务ID |

**响应示例**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "5f2fdc1fd15e421a91db947616f8d2b1",
    "teacherId": 1001,
    "courseId": 2001,
    "title": "第1讲-线性回归",
    "resourceType": "video",
    "status": "success",
    "stage": "上传完成",
    "percent": 100,
    "resourceId": 10001,
    "url": "https://your-bucket.oss-cn-beijing.aliyuncs.com/videos/2026/04/21/xxxx.mp4",
    "objectKey": "videos/2026/04/21/xxxx.mp4"
  }
}
```
