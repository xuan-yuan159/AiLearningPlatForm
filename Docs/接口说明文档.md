# 接口说明文档

本文档用于说明 AI 助教在线学习平台的后端接口协议。

## 全局状态码说明

| 状态码 | 说明 |
| :--- | :--- |
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未授权或 Token 已过期 |
| 500 | 服务器内部错误 |

---

## Knife4j 接口文档

本项目集成了 Knife4j，提供了更美观、易用的接口文档界面。

### 访问地址

| 服务 | 端口 | 访问地址 |
| :--- | :--- | :--- |
| **认证服务 (auth-service)** | 8081 | `http://localhost:8081/doc.html` |

### 调试说明

1. **界面功能**：左侧菜单可以按模块查看接口，支持搜索和离线导出文档。
2. **在线调试**：在接口详情页切换到“调试”页签，填入参数即可发送请求。
3. **全局参数**：可以在右上角的“个性化设置”或“全局参数设置”中统一配置 `Authorization` 请求头。

## 1. 认证模块 (Auth Module)

### 1.1 用户注册

**接口地址**：`/auth/register`

**请求方式**：`POST`

**接口描述**：新用户注册账号。默认注册身份为学生。

#### **请求参数 (JSON)**

| 参数名 | 类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| username | String | 是 | 用户名（唯一） |
| password | String | 是 | 密码 |
| identity | Integer | 否 | 身份（0-学生，1-教师），默认为 0 |
| nickname | String | 否 | 昵称 |
| email | String | 否 | 邮箱 |
| phone | String | 否 | 手机号 |

**请求示例**：
```json
{
    "username": "testuser",
    "password": "password123",
    "identity": 0,
    "nickname": "小明",
    "email": "test@example.com"
}
```

#### **响应参数 (JSON)**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| code | Integer | 状态码（200 表示成功） |
| message | String | 提示信息 |
| data | Object | 响应数据主体（注册成功为 null） |

**响应示例**：
```json
{
    "code": 200,
    "message": "success",
    "data": null
}
```

### 1.2 用户登录

**接口地址**：`/auth/login`

**请求方式**：`POST`

**接口描述**：用户通过用户名和密码进行身份验证。验证成功后，系统将返回用于后续请求鉴权的 JWT Token。

#### **请求参数 (JSON)**

| 参数名 | 类型 | 必选 | 说明 |
| :--- | :--- | :--- | :--- |
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |

**请求示例**：
```json
{
    "username": "admin",
    "password": "password"
}
```

#### **响应参数 (JSON)**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| code | Integer | 状态码（200 表示成功） |
| message | String | 提示信息 |
| data | Object | 响应数据主体 |
| └─ token | String | 身份验证令牌 (JWT) |
| └─ userId | Long | 用户唯一标识 ID |
| └─ role | String | 用户角色 (如: STUDENT, TEACHER, ADMIN) |

**响应示例 (成功)**：
```json
{
    "code": 200,
    "message": "success",
    "data": {
        "token": "eyJhbGciOiJIUzUxMiJ9...",
        "userId": 100,
        "role": "ADMIN"
    }
}
```

**响应示例 (失败)**：
```json
{
    "code": 400,
    "message": "Username and password are required",
    "data": null
}
```

---

### 1.2 用户登出

**接口地址**：`/auth/logout`

**请求方式**：`POST`

**接口描述**：注销当前用户的登录状态，并清除服务端（Redis）中的 Token 缓存。

**请求头要求**：
- `Authorization`: `Bearer {token}`

**响应示例**：
```json
{
    "code": 200,
    "message": "success",
    "data": null
}
```
