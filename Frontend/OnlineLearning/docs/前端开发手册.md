# 前端开发手册

## 1. 项目启动

```bash
cd Frontend/OnlineLearning
npm install
npm run dev
```

## 2. 常用命令

```bash
npm run lint
npm run typecheck
npm run test
npm run test:coverage
npm run build
```

## 3. 目录规范

```text
src/
  api/             接口请求与请求封装
  components/      公共组件与业务组件
  layouts/         页面布局
  router/          路由定义与守卫
  stores/          全局状态管理
  styles/          Sass 变量、Mixin、全局样式
  utils/           工具函数
  views/           页面级组件
tests/unit/        Jest 单元测试
```

## 4. Sass 规范

- 变量集中定义在 `src/styles/_variables.scss`
- Mixin 集中定义在 `src/styles/_mixins.scss`
- 页面样式统一使用 `lang="scss"` 与 BEM 命名
- 断点管理统一使用 `@include down(sm|md|xl)`

## 5. Git 分支策略

- `main`：生产发布分支，仅通过合并请求更新
- `develop`：日常集成分支
- `feature/<模块名>`：功能开发分支，如 `feature/course-management`
- `hotfix/<问题编号>`：线上修复分支
- 合并流程：`feature/hotfix -> develop -> main`

## 6. 代码审查检查表

- 功能是否覆盖需求文档对应场景
- 路由守卫与权限校验是否正确
- 接口异常是否有兜底提示
- Sass 是否复用变量与 Mixin
- 是否补充中文 JSDoc
- ESLint 与 Prettier 是否通过
- 单元测试是否新增并通过
- 覆盖率是否达到 80% 及以上

## 7. Husky 与提交规范

- 已启用 `pre-commit` 钩子：
  - `lint-staged`
  - `npm run test -- --coverage=false`
- 提交前建议先执行：

```bash
npm run lint && npm run test:coverage
```
